О программе

Работа выполнена Сенькиным А.Е. ПИбд-42

Группа: ПИбд-42

Вариант - 3 (Хэш-функция MD5).

MD5 — криптографический хеш-алгоритм, который принимает на входе сообщение произвольной длины и возвращает хеш-значение фиксированной длины, равной 128 битам, обычно представленное в виде 32-значного шестнадцатеричного числа.

Основные шаги алгоритма MD5:

1. Инициализация переменных: Устанавливаются четыре 32-битных переменных (A, B, C, D), которые будут использоваться в процессе вычислений.

2. Предварительная обработка сообщения:

- Дополняется сообщение так, чтобы его длина стала в 64 битах меньше, чем кратное 512.

- Добавляется бит "1" и некоторое количество бит "0" в зависимости от длины сообщения.

Обработка блоков сообщения:

- Сообщение разбивается на блоки по 512 бит.

- Каждый блок подвергается обработке.

Обработка блока:

- Каждый блок разбивается на 16 32-битных слов.

- Выполняются 64 раунда преобразований, в которых используются нелинейные функции, побитовые операции и операции над словами.

Финальная обработка:

Результаты предыдущих операций объединяются и представляют собой 128-битный хеш.

Возможности для сохранения и выбора обрабатываемого файла:

- Пользователь выбирает и загружает файл из системы на сервер

- После обработки файла на сервере, то есть после хеширования файла, файл сохраняется на сервере и пользователь автоматически перенаправляется для скачивания обработанного файла
